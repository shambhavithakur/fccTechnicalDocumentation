Introduction

Shopping and social-media websites and other task-based web applications are expected to retrieve, transfer, and display data in real time. For example, a shopping website must retrieve up-to-date product data from a database server and display the data to users dynamically. You don't want the users to wait until a web developer adds the data to the website code.

If you want to build task-based, dynamic web applications, you need something more than HTML, CSS, and client-side JavaScript, which typically output hard-coded content. You need a suitable server-side technology. Node.js is one such technology. This primer discusses the features and benefits of Node and explains how you can use Node for Windows to perform simple server-side tasks.

Primary Features

Node is a runtime environment that interprets and runs JavaScript code on the server side. Node is both open source and platform independent. Anyone who wants to use Node can download and install it on a standard computer or server operating system (OS).

Because Node runs on an OS directly, it does not support browser-specific JavaScript application programming interfaces (APIs). Instead, it is configured to work with traditional OS APIs, including hypertext transfer protocol (HTTP) and file system libraries.

To enable task automation, Node comes bundled with a core set of files, each of which is preconfigured to perform a specific task. Each file in this core set is called a module. Examples of preconfigured Node modules include the following:

  • http: To transfer data using hyper text transfer protocol (HTTP), use the http module.
  • console: This module makes a simple debugging console available to you.
  • path: If you want to work with file and directory paths, use the path module.

You are not restricted to using the core set of modules, however. You can also employ third-party modules and create your own modules to perform common tasks.

Specific Benefits

Node offers the following benefits to  developers:

  • Familiarity and focus: If you use Node, you can build an entire web application using JavaScript. You don't have to switch between different technologies for the client and server sides.
  • Up-to-date language: JavaScript is continuously being improved. It offers a simplified syntax and code structure and compatibility with recent upgrades to web-server technology. Further, many new and popular languages, such as CoffeeScript and LiveScript, can easily be converted into JavaScript. Therefore, you can also write server-side code for Node in these new languages.
  • Efficiency: You can quickly solve common web-development problems with Node. You can also be assured that applications that you build with Node have optimized throughput and are scalable.
  • Reusability: Numerous reusable modules and libraries are available for Node today, and you can quickly install these modules and libraries using the node package manager (NPM).
  • Extensive support: Node is supported on most web-hosting platforms. In addition, it has a vibrant and helpful community of developers and third-party module providers. This means that you can easily resolve any problems during the installation and use of Node-based web applications.

Installation

To install a stable version of Node on Windows, perform the following steps:

  1. Go to https://nodejs.org/en/.
  2. Select the button that contains the text "Recommended for most users". This will initiate the download of the most recent, recommended version of the Node installation file. This file will be in the Microsoft Installer (MSI) format and have a .msi extension.
  3. In the Save As dialog box that opens, browse to the folder where you want to save the Node installation file, and then click Save.
  4. After the installation file has been downloaded, access the folder were you saved the file and double-click the file.
  5. Follow the default installation prompts to install Node.

An alternative and quicker way to install Node is to use the Chocolatey package manager. To install Node with Chocolatey, perform the following steps:

  1. Install Chocolatey using the instructions available at https://chocolatey.org/install.
  2. Install Node by running the following command from the command line or the PowerShell:
    choco install nodejs-lts

Verifying Your Installation

After you are through with the installation steps, both Node and NPM should be installed on your computer. You can verify these installations by running the following commands at the command prompt separately: 

  • node -v
  • npm -v

Use

You can use the modules of Node to build various types of  applications. For example, you can build a simple web server with the http module.

Building a Web Server

Suppose you want to build a web server named test-node that listens for any kind of HTTP request at http://127.0.0.1:8000/. When a request is received, test-node responds with the string "Hello World". You can perform the following steps to build test-node:

  1. Open the command-line utility.
  2. Run the following command to create a folder named test-node: 
    mkdir test-node
  3. Run the following command to browse to the test-node folder: 
    cd test-node
  4. In a text editor, create a file named hello.js.
  5. In the hello.js file, paste the following code: 
    // Load HTTP module
var http = require("http");
                    
// Create HTTP server and listen on port 8000
http.createServer(function(request, response) {
                    
  // Set response HTTP header
  response.writeHead(200, {'Content-Type': 'text/plain'});
                       
  // Send response text
  response.end('Hello World\n');
}).listen(8000);
                    
// Print URL for accessing server
console.log('Server running at http://127.0.0.1:8000/');
  6. Save the hello.js file in the test-node folder.
  7. At the command prompt, run the following command: 
    node "hello.js"
  8. In a web browser, access http://127.0.0.1:8000/. You should see the text "Hello World" in the upper left corner of the  page that opens.
